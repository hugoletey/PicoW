Projet ISEN4 - Programmation Réseau

Objectifs :

  • Mettre en œuvre la programmation d'une application communiquant via des sockets internet.
  • Mettre en œuvre la problématique du côté client
  • Mettre en œuvre la problématique du côté serveur

Remarque :

  • Les projets sont à réaliser par équipe de 3 à 4 étudiants.
  • On peut répondre aux objectifs du projet en développant à la fois une application serveur et une application client utilisant le même protocole applicatif. Si on souhaite répondre en développant une unique application qui intègre donc les deux types de connexion, on doit se tourner vers le développant d'un proxy (c'est à dire une sorte de relay entre deux applications).
  • Ce proxy, utilisant des protocoles standardisés, on pourra tester son fonctionnement en lien avec des applications clientes et des applications serveurs existantes.

Rendu :

Le projet est à rendre pour le dimanche 12 mars 2023 (avant 23h59). Vous m'envoyer par mail :

  • Le rappel des noms et prénoms des participants au groupe de projet
  • Le code source du projet
  • L'URL YouTube d'une vidéo montrant le fonctionnement du projet

Exemple de sujets :

  • Load balancer HTTP (sur sockets TCP)
  • Proxy HTTP vers FTP (sur sockets TCP)
  • Proxy TFTP (sur sockets UDP)
  • Proxy HTTP d'authentification (sur sockets TCP)
  • Relay SMTP simpliste (sur sockets TCP)
